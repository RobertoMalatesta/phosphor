Optional key

key: string

Inherited from IVNodeAttrs.key

  • Defined in ui/vdom.ts:28

The key id for the node.

If a node is given a key id, the generated DOM node will not be recreated during a rendering update if it only moves among its siblings in the render tree, provided the type of the node does not change.

In general, reordering child nodes will cause the nodes to be completely re-rendered. Keys allow this to be optimized away.

If a key is provided, it must be unique among sibling nodes.
